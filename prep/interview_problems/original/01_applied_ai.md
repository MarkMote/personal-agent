Problem 9. Design Minimal ReAct LLM Agent Loop
-------------------------------------------------
Implement an in-memory **ReAct**-style agent that alternates between **Thought**, **Action**, **Observation** steps until it produces a **Final Answer**.  
Step 2 (“Thought”) must explicitly use **chain-of-thought** formatting.

You must implement the class:

```python
class ReActAgent:
    def __init__(self, llm: Callable[[str], str]): ...
    def run(self, prompt: str, max_steps: int = 5) -> str: ...
```

Agent Contract
--------------
1. `llm` is an **injected** callable that takes a single string (the prompt so far) and returns the next LLM string.
2. The agent loop:
   - **Step 1** append the user prompt.  
   - **Step 2** generate a **Thought** block wrapped in `Chain-of-thought:` … (chain-of-thought reasoning).  
   - **Step 3** generate an **Action** block of the form `Action: <tool_name>(<arg>)`.  
   - **Step 4** execute the tool and append an **Observation** block.  
   - Repeat until the LLM emits a line starting with `Final Answer:` (case-insensitive); return everything after that prefix.  
   - If `max_steps` is exceeded, raise `RuntimeError("max steps")`.

Supported Tools
---------------
- `search(query)` – returns the first sentence of the mock result (see below).  
- `calculate(expr)` – returns `str(eval(expr))` safely (you may assume `expr` is a simple arithmetic string).

Mock Search Results
-------------------
```
search("capital of France") → "Paris is the capital city of France."
search("square root 16")   → "The square root of 16 is 4."
```

Chain-of-Thought Format
-----------------------
The second LLM call (and every Thought step thereafter) must prefix its reasoning with:

```
Chain-of-thought: Let me think step by step.
<free-form reasoning>
```

Constraints
-----------
- All tool calls must be parsed with a single regex: `^Action: (\w+)\((.+?)\)$`.  
- Tool arguments may contain spaces but **not** nested parentheses.  
- Total tokens generated by the LLM must be ≤ 4096 per call (you do **not** need to enforce this).  
- You may assume no concurrent calls.

Example Trace
-------------
```
agent = ReActAgent(llm=my_llm)
answer = agent.run("What is 3 plus the capital of France?")
# Internally produces:
# Thought: I need to find the capital and then add 3.
# Action: search("capital of France")
# Observation: Paris is the capital city of France.
# Thought: Now I add 3 to Paris... wait, I need the numeric value.
# Action: calculate(3 + 0)
# Observation: 3
# Final Answer: 3
```

Skeleton
--------
```python
import re
from typing import Callable

class ReActAgent:
    def __init__(self, llm: Callable[[str], str]):
        self.llm = llm

    def run(self, prompt: str, max_steps: int = 5) -> str: ...

    def _execute(self, action: str, arg: str) -> str: ...
```

(You only need to implement the agent loop, tool dispatcher, and CoT formatting; the LLM itself is mocked via injection.)

Problem 10. Design “Hello-World” MCP Server

Implement a minimal **Model-Context-Protocol** (MCP) server that exposes one tool, `hello`, and returns a greeting.  
The server must speak **stdio JSON-RPC** and advertise the MCP version it implements.

You must implement:

```python

```

Requirements
------------
1. Accept **stdin** and write **stdout** only; no other I/O.  
2. Support two MCP methods:  
   - `mcp.initialize` – reply with protocol version and capabilities.  
   - `mcp.tools/call` – expose a single tool `hello` that takes `{name: str}` and returns `{greeting: str}`.  
3. Ignore notifications (`mcp.initialized`).  
4. Log **only** to **stderr** (one line per request for debugging).  
5. Exit cleanly on EOF or malformed JSON.  
6. Use only the Python standard library.

Message Framing
---------------
- Each JSON-RPC message is sent as **raw JSON** followed by **two newlines** `\n\n`.  
- Read messages with `sys.stdin.buffer.readline()` until `\n\n`.

Capabilities Advertised
-----------------------
```json
{
  "protocolVersion": "2024-11-05",
  "capabilities": { "tools": true },
  "serverInfo": { "name": "hello-mcp", "version": "0.1.0" }
}
```

Tool Schema
-----------
```json
{
  "name": "hello",
  "description": "Say hello",
  "inputSchema": {
    "type": "object",
    "properties": { "name": { "type": "string" } },
    "required": ["name"]
  }
}
```

Example Exchange
----------------
```
→ {"jsonrpc":"2.0","id":1,"method":"mcp.initialize","params":{}}

← {"jsonrpc":"2.0","id":1,"result":{"protocolVersion":"2024-11-05","capabilities":{"tools":true},"serverInfo":{"name":"hello-mcp","version":"0.1.0"}}}

← {"jsonrpc":"2.0","method":"mcp.tools/list","params":{}}

← {"jsonrpc":"2.0","id":2,"result":{"tools":[{"name":"hello","description":"Say hello","inputSchema":{"type":"object","properties":{"name":{"type":"string"}},"required":["name"]}}]}}

→ {"jsonrpc":"2.0","id":3,"method":"mcp.tools/call","params":{"name":"hello","arguments":{"name":"Alice"}}}

← {"jsonrpc":"2.0","id":3,"result":{"greeting":"Hello, Alice!"}}
```

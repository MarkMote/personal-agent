## How to practice
1. Open a blank CoderPad (coderpad.io/sandbox) or plain editor
2. Set a 45-minute timer
3. Talk out loud the ENTIRE time
4. Stage 1 should be done in 10-15 min max
5. After each stage, ask: "What's wrong with this? What breaks at scale?"
6. It's fine to not finish. Getting through Stage 2 cleanly > rushing to Stage 4.

## The pattern
Every problem follows the same arc:
- **Stage 1:** Get the basic thing working. Clean, simple, correct.
- **Stage 2:** Handle a complication (edge cases, new requirement, performance issue).
- **Stage 3:** Major extension (new data structure, algorithm upgrade, feature).
- **Stage 4-5:** Stretch goals. Don't worry if you don't reach these.
